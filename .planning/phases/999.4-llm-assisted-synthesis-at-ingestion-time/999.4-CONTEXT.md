# Phase 999.4: LLM-Assisted Synthesis at Ingestion Time - Context

**Gathered:** 2026-06-18
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase delivers an automated entity and relationship description synthesis step during document ingestion. When new documents are parsed, entity nodes in LanceDB are enriched with synthesized prose summaries. v1 leaves ports by adding empty placeholders/nullable columns for `summary`, `summary_vector`, and `unsummarized_refs` in the LanceDB node table, along with a pass-through async Tokio background loop.

</domain>

<decisions>
## Implementation Decisions

### LLM Calling Strategy
- **D-01:** Implement **Asynchronous Background Processing**. Extracted entities are immediately saved to LanceDB, and their summaries are queued and processed asynchronously in the background by a rate-limited worker thread/Tokio task in Rust. This prevents LLM latency from blocking document upload API responses.

### Storage Schema & Vector Embeddings
- **D-02:** Use a **Double Vector Embedding** approach. Separate vector columns in LanceDB will store `name_vector` (for exact and synonym lookup of entity nodes) and `summary_vector` (for semantic matching of entity descriptions).
- **D-03:** In v1, the LanceDB `nodes` table schema will register `summary` (nullable Text) and `summary_vector` (nullable Float Array) to ensure full forward-compatibility.

### Summary Update & Drift Strategy
- **D-04:** Implement **Threshold-based Accumulation & Rewrite**. When new documents reference an existing entity, we store the reference chunk ID in an `unsummarized_refs` array on the node. Once the count exceeds a threshold (e.g., 5 new references), a background worker job is triggered to update and rewrite the summary.
- **D-05:** In v1, the `nodes` table schema will register the `unsummarized_refs` column as an empty/nullable array.

### Agent Discretion
- The agent has discretion over the default threshold count for triggering rewrites, the queue batch size, and the configuration of the Tokio background worker channel.

</decisions>
