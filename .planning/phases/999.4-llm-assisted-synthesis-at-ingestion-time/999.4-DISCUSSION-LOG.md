## Ingestion Latency / LLM Calling Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Option A | Synchronous (Blocked Ingestion) — Call the LLM inline during ingestion. | |
| Option B | Asynchronous (Background Job) — Extract nodes immediately, queue summary generation asynchronously. | ✓ |
| Option C | Lazy/On-Demand — Generate summaries on first query and cache them. | |

**User's choice:** Option B (Asynchronous background processing)
**Notes:** To showcase strong backend systems design to recruiters, we avoid synchronous blocking LLM calls. We will save nodes immediately and process their summaries asynchronously in the background.

## Storage Schema & Vector Embeddings

| Option | Description | Selected |
|--------|-------------|----------|
| Option A | Double Vector Embedding — Separate vector columns for name and summary description. | ✓ |
| Option B | Combined Single Vector — Concatenate name and summary, generating one embedding. | |
| Option C | Name-only Vector — Embed name only, store summary as text metadata. | |

**User's choice:** Option A (Double Vector Embedding)
**Notes:** Store name and summary separately, using distinct vector columns in LanceDB (`name_vector` and `summary_vector`). This avoids diluting exact entity matching with long prose descriptions in a single vector field.

## Summary Update & Drift Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Option A | Threshold-based Accumulation & Rewrite — Batch update summaries after N references. | ✓ |
| Option B | Continuous Inline Merge — Merge immediately on every write. | |
| Option C | Static (Write-Once) — Summarize once upon first discovery, never update. | |

**User's choice:** Option A (Threshold-based Accumulation & Rewrite)
**Notes:** Store document references in an array metadata column on the node, and rewrite/update summaries in batches when references exceed a threshold (e.g., 5 new references). This demonstrates cost awareness and robust batch-scheduling design.
