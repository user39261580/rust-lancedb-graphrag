# Phase 999.5: Compile-Time Semantics on Graph Nodes - Context

**Gathered:** 2026-06-19
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase delivers a compile-time semantics execution layer for GraphRAG query traversal. Instead of evaluating raw source chunks at query time, the traverser reads pre-computed node and edge summaries generated during ingestion. v1 leaves ports by establishing separate `nodes` and `edges` tables in LanceDB with nullable description/vector fields, and implementing a fallback-enabled context assembly interface in Rust.

</domain>

<decisions>
## Implementation Decisions

### Summary Granularity
- **D-01:** Summarize at both the **Node & Edge Level**. Synthesize descriptions for individual entity nodes as well as relationship edges (connections) that have multiple source citations, storing summaries on both.

### Context Assembly Strategy
- **D-02:** Implement a **Fallback-Enabled** context assembly strategy. The traverser attempts to retrieve and inject pre-computed summaries. If a summary is missing (e.g. background job still pending), it automatically falls back to fetching raw text chunks.
- **D-03:** In v1, the context assembler in Rust will support this dual strategy. Since summaries will be empty in v1, it will gracefully fall back to returning raw source chunks.

### Relationship Schema in LanceDB
- **D-04:** Use a **Separate Edges Table** in LanceDB. Store nodes in a `nodes` table and relationships in a dedicated `edges` table.
- **D-05:** The `edges` table schema in v1 will include: `id` (UUID), `source_node_id` (UUID), `target_node_id` (UUID), `relationship_type` (String), `summary` (nullable Text), and `summary_vector` (nullable Float Array).

### Agent Discretion
- The agent has discretion over the exact prompt templates for summarizing edges, the retrieval thresholds for fallback triggers, and internal cache policies for traversed edges.

</decisions>
