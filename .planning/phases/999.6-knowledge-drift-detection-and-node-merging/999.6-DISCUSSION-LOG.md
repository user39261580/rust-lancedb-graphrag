## Data Mutation Policy

| Option | Description | Selected |
|--------|-------------|----------|
| Option A | Non-Destructive Linking — Create SAME_AS edges, preserve nodes and citations. | ✓ |
| Option B | Control Plane Alerts & Approval Queue — Emit events, track duplicates in Postgres/Go. | |
| Option C | Destructive Merging — Delete duplicates, overwrite nodes. | |

**User's choice:** Option A (Non-Destructive Linking)
**Notes:** To build a product-grade, auditable application, we must maintain data/citation provenance. Merging or overwriting nodes is destructive and dangerous. We will create `SAME_AS` edges in the `edges` table to link duplicates, resolving them at query time.
